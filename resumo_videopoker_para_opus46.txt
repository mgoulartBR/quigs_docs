Declaração correta da variável:

Var shCards:Sheet

Carregamento do sheet:
----------------------
Function Form1.Open.Event:Void()
    Randomize Time.Second.Get * Time.Minute.Get

    ; Carregar o sprite sheet manualmente em runtime
    shCards.Load App.Path+"cards.qs2"

    ; Show card backs initially
    For i=0 To 4
        If i=0 Then
            CardImg1.Tile.Set shCards,TILE_BACK
            CardImg1.Refresh
        EndIf
        If i=1 Then
            CardImg2.Tile.Set shCards,TILE_BACK
            CardImg2.Refresh
        EndIf
        If i=2 Then
            CardImg3.Tile.Set shCards,TILE_BACK
            CardImg3.Refresh
        EndIf
        If i=3 Then
            CardImg4.Tile.Set shCards,TILE_BACK
            CardImg4.Refresh
        EndIf
        If i=4 Then
            CardImg5.Tile.Set shCards,TILE_BACK
            CardImg5.Refresh
        EndIf

        gSlot=i
        gShow=0
        ShowHold
    Next

    LblResult.Text.Set "Welcome to Video Poker!"
    LblResult.Refresh
    UpdateDisplay
    gamePhase=0
EndF

    If gamePhase=0 Then
        bet=bet+1

            bet=5
        EndIf
        UpdateDisplay

    If gamePhase=2 Then

HIPÓTESES PARA OS PROBLEMAS
================================================================================

PARA AS CARTAS NÃO APARECEREM:
-------------------------------
1. O sheet não está sendo carregado - shCards.Load falha silenciosamente
2. Os controles Image não estão configurados no Form Designer - faltam propriedades
3. Tile numbers errados - TILE_BACK=13 pode não existir no cards.qs2
4. Formato do arquivo errado - cards.qs2 pode estar corrompido ou incompatível
5. Sintaxe de Tile.Set incorreta - pode precisar de outra abordagem
6. Os controles Image podem precisar ter Sheet e Tile inicializados no Form Designer
7. Pode ser necessário usar IncludeSheet ao invés de Load runtime

PARA OS BOTÕES NÃO FUNCIONAREM:
--------------------------------
1. Eventos não estão ligados no Form Designer - falta conectar eventos aos controles
2. gamePhase impede execução - mas deveria funcionar no início (gamePhase=0)
3. Os controles Button podem ter nomes ligeiramente diferentes no Form Designer
4. Pode ser necessário usar outra sintaxe para eventos de Button
