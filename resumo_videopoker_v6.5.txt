COMANDOS DESCOBERTOS NO MANUAL (manual.txt)

SHEET MANAGEMENT (linhas 425-430 do manual):
- Sheet.Load Sheet%,SheetFile$     → Carrega sheet
- Sheet.Free Sheet%                → Libera memória
- Sheet.Status% Sheet%             → 0=Empty, 1=In use, >99=Error
- Sheet.Tiles% Sheet%              → Número de tiles
- Sheet.Size% Sheet%               → Tamanho em bytes
- Sheet.Colour.Depth% Sheet%       → 0=4 cores, 1=16 cores

IMAGE CONTROL (linha 92):
- Ctrl.Image.Tile.Set Control%,Sheet%,Tile%  → Set tile
- Ctrl.Refresh Control%                       → Redraw control
- Ctrl.GetID%(Form,DisplayPos)               → Get control ID

TOOL IMAGE (linha 259):
- Tool.ImgBox.Tile.Set Control%,Sheet%,Tile% → Para Tool controls

NOTA: Manual mostra Sheet.Load Sheet%,SheetFile$ (2 params)
vs código usa shCards.Load "file" (method style)
→ Ambos podem funcionar pois 2048 usa method style

HIPÓTESES RESTANTES (ORDENADAS POR PROBABILIDADE)
================================================================================

1. **SHEET NÃO CARREGOU DE VERDADE**:
   .Load não deu erro mas Sheet.Status pode ser 0 (empty) ou >99 (error)
   File.Exist=TRUE não garante que .Load funcionou
   → v6.5 TESTA com Sheet.Status e Sheet.Tiles

2. **CONFLITO IncludeSheet + .Load**:
   Ambos tentam usar o mesmo sheet container
   → Se Sheet.Status=0, tentar REMOVER IncludeSheet

3. **TAMANHO DO CONTROLE ERRADO**:
   Width=16 mas display width=32 (pixel ratio 2:1)
   → Testar com Width=32 no Form Designer

4. **Ctrl.Image.Tile.Set RECEBE Int MAS shCards É Sheet**:
   Manual diz Sheet% (Int), código passa Var:Sheet
   → Testar com Var shCards:Int e Sheet.Load shCards,"file"

5. **IMAGEBOX PRECISA DE CONFIGURAÇÃO ADICIONAL**:
   Talvez precise de Form1.Control.Refresh ao invés de Ctrl.Refresh
   → Explorer usa Form1.Control.Refresh após setar tiles
